# Migration Context

This directory holds per-module migration artifacts generated by the Migration Coordinator pipeline.

## Structure

Each module being migrated gets its own subdirectory:

```
context/migration/
├── manifest.md                   # Progress dashboard for all modules
├── {module-name}/
│   ├── discovery-report.md       # Phase 1: Business rules, endpoint map, data flow
│   ├── migration-plan.md         # Phase 2: Strategy, API contracts, component tree
│   ├── gap-report.md            # Phase 2: Completeness validation gaps
│   ├── bugs-to-fix.md           # Phase 2: Jira bugs (fix now vs defer)
│   ├── api-gap-report.md        # Phase 3: Missing endpoint contracts + MSW mocks
│   ├── backend/                  # Phase 4: Backend migration artifacts
│   ├── frontend/                 # Phase 5: Frontend migration artifacts
│   ├── validation/               # Phase 6: Parity reports
│   └── cutover/                  # Phase 7: Deployment & rollback plans
```

## Workflow

1. `@Migration Coordinator migrate module:{name}` — starts the pipeline
2. `@Migration Coordinator status` — shows progress
3. `@Migration Coordinator resume module:{name}` — resumes from last checkpoint
